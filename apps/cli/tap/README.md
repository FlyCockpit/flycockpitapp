# Homebrew tap

This folder documents how the `cockpit` Homebrew formula is published. The
actual tap is the separate GitHub repository:

```text
https://github.com/FlyCockpit/homebrew-tap
```

Users install from that tap with:

```sh
brew install flycockpit/tap/cockpit
```

## One-time setup

1. Keep `FlyCockpit/homebrew-tap` public so Homebrew can clone it without auth.
2. In `FlyCockpit/flycockpitapp`, create a protected `release` environment
   and add an environment secret named `HOMEBREW_TAP_TOKEN`. Use a
   fine-grained GitHub token with `contents:write` access to
   `FlyCockpit/homebrew-tap`. The default `GITHUB_TOKEN` cannot push to a
   different repository.

## Release behavior

The root `Release CLI` workflow in this repository does the full publish path.
The Homebrew publish job uses the protected `release` environment:

1. Builds cross-platform `cockpit` artifacts with `dist`.
2. Creates the GitHub Release and uploads installers, checksums, archives, and
   the generated `cockpit.rb` formula.
3. Checks out `FlyCockpit/homebrew-tap`, copies `cockpit.rb` to
   `Formula/cockpit.rb`, commits it, and pushes to `main`.

The formula is generated by cargo-dist from `dist-workspace.toml` (the
`formula = "cockpit"` override names it after the binary rather than the
`cockpit-cli` package); do not edit it by hand. If the tap name changes, update
`tap = "FlyCockpit/homebrew-tap"` in `dist-workspace.toml` and review the root
release workflow.

## How users install

```sh
brew install flycockpit/tap/cockpit
brew upgrade cockpit
```

Private repositories do not work with the one-line `brew install` flow unless
the user has GitHub/Homebrew auth configured. Keep both the source repo and the
tap public for the smooth install path.
